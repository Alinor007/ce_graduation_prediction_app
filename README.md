# CE Graduation Predictor

A machine learning web application that predicts whether a Civil Engineering student at **Mindanao State University – Main Campus, Marawi City** will graduate on time. Built as an undergraduate thesis project for the CE Department, College of Engineering.

---

## Overview

The app uses three separate ML models — each targeting a different point in a student's academic journey — to give advisers and faculty an early, data-driven signal about graduation risk. All predictions are decision-support only and should always be combined with human judgment.

| Model | When to Use | Features Used |
|---|---|---|
| **Model 1 — Pre-Admission** | At admission / orientation | SASE scores + SHS GWA + SHS strand |
| **Model 2 — In-Program** | Each semester after grades post | Subject grades + CGPA + retake history |
| **Model 3 — Combined** | When full profile is available | All of the above combined |

Each model runs two algorithms — **Random Forest** and **Logistic Regression** — and displays both results side-by-side for comparison.

---

## Features

- **Three prediction models** covering pre-admission, in-program, and combined data
- **Philippine grading scale** (1.0–5.0) with "not taken" handling via automatic imputation
- **Risk classification** — Low / Moderate / High based on on-time probability
- **Algorithm comparison chart** — RF vs Logistic Regression on the same input
- **Feature importance chart** — shows which features the Random Forest weighted most
- **Responsive design** — maroon × gold MSU-CE identity, works on desktop and mobile
- **Sidebar algorithm selector** — switch between RF and LR without re-entering inputs

## Setup

### Requirements

- Python 3.10+
- Streamlit
- scikit-learn
- pandas, numpy
- matplotlib
- joblib

Install all dependencies:

```bash
pip install streamlit scikit-learn pandas numpy matplotlib joblib
```

### Generating the Model Files

The `model/` folder is **not included** in the repository — you must generate it from the training notebook first.

1. Open `CE_Graduation_Prediction_v3.ipynb` in Jupyter.
2. Run all cells to train the models.
3. Add this export block at the end and run it:

```python
import joblib, os
os.makedirs('model', exist_ok=True)

# Model 1 — SASE + GWA + Strand
joblib.dump(models_1['Random Forest'],       'model/model_1_rf.pkl')
joblib.dump(models_1['Logistic Regression'], 'model/model_1_lr.pkl')
joblib.dump(imp1, 'model/imputer_1.pkl')
joblib.dump(scl1, 'model/scaler_1.pkl')

# Model 2 — Critical Grades + CGPA + Retakes
joblib.dump(models_2['Random Forest'],       'model/model_2_rf.pkl')
joblib.dump(models_2['Logistic Regression'], 'model/model_2_lr.pkl')
joblib.dump(imp2, 'model/imputer_2.pkl')
joblib.dump(scl2, 'model/scaler_2.pkl')

# Model 3 — Combined
joblib.dump(models_3['Random Forest'],       'model/model_3_rf.pkl')
joblib.dump(models_3['Logistic Regression'], 'model/model_3_lr.pkl')
joblib.dump(imp3, 'model/imputer_3.pkl')
joblib.dump(scl3, 'model/scaler_3.pkl')

print("✅ All 12 files saved to model/")
```

4. Copy the `model/` folder next to `app.py`.

### Running the App

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501` by default.

## How It Works

### Data Flow

```
User Input (Streamlit widgets)
        ↓
   tabs.py  — builds input dict, selects model bundle
        ↓
predictor.py — imputes missing grades → scales → predicts
        ↓
 results.py  — renders banner, gauge, comparison chart, importance
```

### Preprocessing

Each model bundle contains a fitted `SimpleImputer` and `StandardScaler` saved from the training notebook. When a student hasn't taken a subject yet, the grade field is left blank (encoded as `NaN`) and the imputer fills it with the training-set mean for that subject. The imputer is applied manually via `_safe_impute()` in `predictor.py` to avoid sklearn version mismatch errors between the training environment and the app.

### Risk Classification

On-time probability thresholds used by `PredictionResult.risk_level`:

| Threshold | Risk Level |
|---|---|
| ≥ 70% | 🟢 Low |
| 45%–69% | 🟡 Moderate |
| < 45% | 🔴 High |

### Subject Grading Scale

The app uses the Philippine university grading scale:

| Grade | Meaning |
|---|---|
| 1.0 | Highest |
| 1.25, 1.5, 1.75 | Above average |
| 2.0, 2.5, 2.75 | Average |
| 3.0 | Passing |
| 5.0 | Failed |
| — Not taken — | Subject not yet attempted (treated as NaN) |

### SHS GWA Bands

Senior High School GWA is encoded as a numeric midpoint of the band selected:

| Band | Encoded Value |
|---|---|
| 80–85 | 82.5 |
| 81–85 | 83.0 |
| 86–90 | 88.0 |
| 91–93 | 92.0 |
| 91–95 | 93.0 |
| 96–99 | 97.5 |
| 96–100 | 98.0 |

## Module Reference

### `app.py`
Entry point. Calls `st.set_page_config()`, injects CSS, guards against missing model files, loads models, renders the sidebar, and routes to the four tabs.

### `tabs.py`
One function per tab (`render_model1_tab`, `render_model2_tab`, `render_model3_tab`). Each function collects inputs via `ui_components`, builds the input dict, calls `predictor.predict()`, and passes results to `ui_components.render_results()`. Contains `_strand_to_onehot()` for one-hot encoding the SHS strand.

### `ui_components.py`
Thin re-export facade. Imports everything from `styles`, `layout`, `inputs`, `results`, and `about` and exposes them under a single namespace so `tabs.py` and `app.py` don't need to know about the internal split.

### `styles.py`
All CSS (design tokens, layout, components, animations, dark sidebar) as a single `CSS` string injected via `st.markdown(..., unsafe_allow_html=True)`. Also holds all HTML template strings (`NAVBAR_HTML`, `HERO_HTML`, `MODELS_OVERVIEW_HTML`, `RESULT_HTML`, `PLACEHOLDER_HTML`) and the `PH_GRADE_OPTIONS` / `PH_GRADE_LABELS` constants.

### `layout.py`
Page-structure functions: `inject_css()`, `render_hero()`, `render_models_overview()`, `render_sidebar()`, `card_open()`, `card_close()`, `grade_note()`. None of these deal with student data.

### `inputs.py`
Four input-collection functions used by `tabs.py`:
- `sase_inputs(prefix)` — four SASE sub-scores + auto-computed total
- `shs_inputs(prefix)` — GWA band selector + strand selector
- `grade_inputs(prefix)` — per-subject selectboxes with "not taken" option
- `academic_standing_inputs(prefix, grades)` — CGPA slider + retake/fail counts (auto-suggests from grade dict)

### `results.py`
Renders prediction output: result banner (green/red), probability gauge, algorithm comparison bar chart, risk interpretation, and RF feature importance chart. All `st.pyplot()` calls use `width="stretch"`.

### `about.py`
Self-contained about page with thesis title card, author cards, adviser card, and institution block.

### `predictor.py`
Pure Python — no Streamlit. Defines `PredictionResult` and `AlgorithmResult` dataclasses. Core functions:
- `predict(bundle, inputs, features, algorithm)` — single prediction
- `predict_all_algorithms(bundle, inputs, features)` — runs all algorithms for comparison
- `get_feature_importances(bundle, features)` — extracts RF importances sorted ascending

### `model_loader.py`
Loads all 12 `.pkl` files from the `model/` directory using `joblib`. Decorated with `@st.cache_resource` so files are only read from disk once per session. `missing_files()` is called at startup — the app halts with instructions if any file is absent.

### `charts.py`
Three `matplotlib` figure builders, each returning a `Figure` object:
- `gauge(prob_ontime)` — thin horizontal bar with 50% threshold line
- `algorithm_comparison(results)` — grouped bar chart, green/red by outcome
- `feature_importance(pairs)` — horizontal bar chart, top-3 features highlighted in gold

### `config.py`
Single source of truth for all constants: feature column names per model, human-readable feature labels, GWA band lookup table, SHS strand list, model file registry, and the notebook export code snippet.

### `save_models.py`
Standalone script / notebook cell that exports trained sklearn objects to `model/*.pkl`. Run this once from the training notebook after training is complete.
